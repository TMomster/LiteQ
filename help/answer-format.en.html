LiteQ Answer Data Format Guide

This guide explains how to fill in survey answer data based on actual survey results

Data Overview

Answer data consists of two parts: the total number of participants and the count of responses for each option of each question.

All values must be non-negative integers (counts cannot be negative or decimal).

Per-Question Entry (Manual)

In Step 2 of LiteQ ("Result Data"), the system automatically generates input forms based on the survey structure:

  • First, enter the total number of participants
  • Each option for each question will have a corresponding input field
  • Single Choice (sc): Shows remaining count for that question; sum of all options must not exceed the total
  • Multiple Choice (mc): Each option independently shows available count; a single option's count must not exceed the total
  • Open-ended (a): No data input needed

Batch Import (Quick Way)

If you have existing statistics, click the "Parse & Fill" button and paste data using the following format:

total: total_count
q1_data: opt1_count opt2_count opt3_count ...
q2_data: opt1_count opt2_count ...
Example: Commute Mode Survey
total: 150
20 30 60 40
50 25 75
30 25 20 60 15

Format Rules:

  • First line total: count specifies the total number of participants
  • Each subsequent line corresponds to one question (in survey order)
  • Each line contains space-separated numbers corresponding to option order
  • Lines for open-ended questions (a) can be omitted
  • The number of values per line must match the number of options for that question

Validation Rules

  • Total must be greater than 0
  • All counts must be non-negative integers
  • For single-choice questions, the sum of all options must not exceed the total
  • For multiple-choice questions, a single option's count must not exceed the total
  • For multiple-choice questions, the sum of all options may exceed the total (since one person can select multiple options)
  • The number of rows and columns must match the survey structure exactly

FAQ

Q: Can the sum of single-choice options be less than the total?

A: Yes. The single-choice sum should be ≤ total, allowing for unanswered questions. However, if the sum exceeds the total, an error will be reported.

Q: Is it normal for the sum of multiple-choice options to exceed the total?

A: Yes. In a multiple-choice question, each person can select multiple options, so the total number of selections may exceed the number of participants.

Q: How do I fill in data for open-ended questions?

A: Open-ended questions do not require numerical data. The system will automatically label them as "Open-ended."
